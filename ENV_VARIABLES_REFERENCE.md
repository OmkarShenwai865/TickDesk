# Environment Variables Reference

Quick reference for all environment variables needed for deployment.

## 🔐 Backend (Render) - Environment Variables

### Required Variables

```bash
# Django Settings
DJANGO_SETTINGS_MODULE=config.settings_production
SECRET_KEY=<auto-generated-by-render-or-use-django-secret-key-generator>
DEBUG=False
ALLOWED_HOSTS=your-app-name.onrender.com

# Database (Neon PostgreSQL)
DATABASE_URL=postgresql://user:password@ep-xxx.region.aws.neon.tech/neondb?sslmode=require

# CORS (Important!)
CORS_ALLOWED_ORIGINS=https://your-frontend.vercel.app,http://localhost:5173

# Cloudinary (File Storage)
CLOUDINARY_CLOUD_NAME=dxxxxx
CLOUDINARY_API_KEY=123456789012345
CLOUDINARY_API_SECRET=abcdefghijklmnop
```

### Optional Variables (Email)

```bash
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-gmail-app-password
DEFAULT_FROM_EMAIL=your-email@gmail.com
PLATFORM_NOTIFY_EMAIL=admin-email@example.com
```

## 🌐 Frontend (Vercel) - Environment Variables

```bash
VITE_API_URL=https://your-backend.onrender.com/api/
```

**Important**: The URL must end with `/api/`

## 📋 How to Get Each Value

### 1. SECRET_KEY
Generate using:
```python
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```
Or use: https://djecrety.ir/

### 2. DATABASE_URL
- From Neon dashboard → Connection Details → Connection string
- Format: `postgresql://user:password@host/database?sslmode=require`

### 3. ALLOWED_HOSTS
- Your Render backend URL (without https://)
- Example: `tickdesk-backend.onrender.com`

### 4. CORS_ALLOWED_ORIGINS
- Your Vercel frontend URL (with https://)
- Example: `https://tickdesk-xyz.vercel.app,http://localhost:5173`
- Separate multiple URLs with commas (no spaces)

### 5. Cloudinary Credentials
- From Cloudinary Dashboard → Settings → Access Keys
- Cloud Name: Found on dashboard
- API Key: Found on dashboard
- API Secret: Click eye icon to reveal

### 6. Email Settings (Optional)
**For Gmail:**
1. Enable 2-Factor Authentication on your Google account
2. Go to: https://myaccount.google.com/apppasswords
3. Generate "App Password" for "Mail"
4. Use that password (16 characters, no spaces)

**Alternative free email services:**
- SendGrid (100 emails/day free)
- Mailgun (100 emails/day free)
- AWS SES (62,000 emails/month free)

## 🔄 Environment Variable Priority

1. **Render** reads from: Dashboard → Environment tab
2. **Vercel** reads from: Project Settings → Environment Variables
3. **Local Development** reads from: `.env` files

---

## ⚠️ Important Notes

1. **Never commit** `.env` files to GitHub
2. `.env.example` files are safe to commit (they show structure, not values)
3. Update `CORS_ALLOWED_ORIGINS` after deploying frontend
4. Update `ALLOWED_HOSTS` with your actual Render domain
5. Database URL must include `?sslmode=require` for Neon

---

## ✅ Verification Checklist

Before going live, verify:

- [ ] All required backend variables are set in Render
- [ ] VITE_API_URL is set in Vercel
- [ ] SECRET_KEY is strong and unique (not the default)
- [ ] DEBUG is set to False in production
- [ ] CORS includes your Vercel URL
- [ ] ALLOWED_HOSTS includes your Render domain
- [ ] Cloudinary credentials are correct (test upload)
- [ ] Database URL connects successfully
- [ ] Email settings work (send test email)

## 🐛 Testing Environment Variables

### Test Backend Connection
```bash
curl https://your-backend.onrender.com/admin/
# Should return Django admin login page
```

### Test Frontend API Connection
Open browser console on your Vercel site and run:
```javascript
console.log(import.meta.env.VITE_API_URL)
// Should show: https://your-backend.onrender.com/api/
```

### Test Cloudinary
Upload a file through your app and check:
- Cloudinary Dashboard → Media Library
- File should appear there

## 🔧 Common Issues

**CORS errors in browser console:**
- Check `CORS_ALLOWED_ORIGINS` includes your Vercel URL
- Must start with `https://` (no trailing slash)

**Database connection fails:**
- Verify `?sslmode=require` is in DATABASE_URL
- Check Neon database is not paused

**Static files not loading:**
- Run `python manage.py collectstatic` (done in build script)
- Check `STATIC_ROOT` and `STATIC_URL` in settings

**Cloudinary uploads fail:**
- Verify all 3 credentials (name, key, secret)
- Check API key has upload permissions
